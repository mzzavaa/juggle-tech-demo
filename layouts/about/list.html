How to Fork and Deploy Your Own JuggleTech

1

Fork the Repository

Go to the JuggleTech GitHub repository and click the "Fork" button in the top-right corner.

2

Enable GitHub Pages

Go to your repository's Settings > Pages and select "GitHub Actions" as the source.

3

Customize Your Dashboard

Edit the following files to personalize your dashboard:

  • hugo.toml - Update your GitHub username and site title
  • data/modules.json - Track your learning modules
  • data/achievements.json - Update your achievements
  • data/practice.json - Log your practice sessions
4

Commit and Push Changes

After making your changes, commit and push them to your repository. The GitHub Action will automatically build and deploy your site.

5

Access Your Dashboard

Your personalized JuggleTech dashboard will be available at https://yourusername.github.io/juggletech/

Contributing to JuggleTech

We welcome contributions to the JuggleTech project! If you've created courses or use cases that you think would benefit others, here's how you can contribute:

Create a Branch

Create a new branch in your fork for your feature or content.

git checkout -b feature/my-new-courses

Make Your Changes

Add new courses to data/modules.json or new achievements to data/achievements.json.

Submit a Pull Request

Push your branch to your fork and submit a pull request to the main repository.
